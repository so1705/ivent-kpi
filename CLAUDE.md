## 1. プロジェクトの概要
`ivent-kpi` は、テレアポや営業イベントのパフォーマンス（KPI）をリアルタイムで管理・分析するためのWebアプリケーションです。スタッフごとの架電数、本人接続数、アポイント獲得数などの指標を追跡し、チーム全体の目標達成を支援します。

主な機能：
- **ダッシュボード**: 個人およびチームの目標達成率を視覚化。
- **実績報告**: スタッフが日々の数値を簡単に入力できるフォーム。
- **分析・AIアドバイス**: 蓄積されたデータに基づき、ボトルネックの特定と改善策を提案。
- **シフト・勤怠管理**: メンバーの出勤予定と実績に基づいた人件費計算。
- **システム管理**: 案件（イベント）の作成、メンバー登録、目標数値の設定。

## 2. 使用している技術スタック
- **Frontend**: React 19 (Vite)
- **Styling**: Tailwind CSS (v4系)
- **Backend/Database**: Firebase (Firestore, Authentication - Googleログイン)
- **Icons**: Lucide-react (SVGコンポーネント化して利用)
- **Export**: html2canvas, jspdf (レポート出力用)
- **その他**: Google Apps Script (GAS) 連携（スプレッドシート同期）

## 4. 主要なファイルの役割
- **`src/App.jsx`**: アプリケーションの心臓部です。以下のすべてが含まれています：
  - Firebaseとのリアルタイム同期（`onSnapshot`）。
  - ユーザー認証の状態管理。
  - 各画面（Dashboard, Analytics, ShiftView, AttendanceView, Settings）の実装。
  - 定数定義（アイコンデータ、ラベル、初期目標値）。
- **`tailwind.config.js`**: モダンでプレミアムなデザインを実現するためのカラーパレットやユーティリティの設定。

## 5. 開発の進め方・ルール
- **コンポーネント構成**: 現在は開発効率を重視し、`App.jsx` 内に主要なコンポーネントをフラットに定義しています。規模が拡大した場合は `src/components/` への分割を検討します。
- **状態管理**: Reactの `useState`, `useEffect`, `useMemo` を活用し、FirebaseのデータをリアルタイムでUIに反映させます。
- **コードスタイル**:
  - 関数・変数は `camelCase`。
  - コンポーネント名は `PascalCase`。
  - 定数は `UPPER_SNAKE_CASE`。
- **コメント**: 複雑なロジックやFirebaseのパス指定には日本語のコメントを付与します。
